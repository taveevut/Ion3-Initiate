Lazy loading

อะไรคือ Lazy Loading

หลักการทำงานของ Lazy Loading ยกตัวอย่างให้เข้าใจง่ายๆ คือ อะไรที่ยังไม่ถูกใช้งาน ก็ยังไม่ต้องโหลด ยกตัวอย่างเช่น ถ้าผู้ใช้งานยังเลื่อนมาไม่ถึงจุดที่มีรูปอยู่ รูปนั้นก็จะยังไม่ถูกโหลดขึ้นมาแสดงผล นั่นทำให้เว็บของเราโหลดเฉพาะสิ่งที่จำเป็นขึ้นมาใช้งานก่อน นอกจาก Lazy Loading จะทำงานกับรูปภาพ และมีเดียต่างๆ แล้ว ยังทำงานกับ script ได้อีกด้วย หากท่านเป็นนักพัฒนาเว็บไซต์หรือแอปพลิเคชันด้วยเฟรมเวิร์ค angular คงคุ้นชินกับ module และ component เป็นอย่างดี ปัญหาคือ เมื่อแอปพลิเคชันของเรามีขนาดใหญ่ขึ้น มีหลายหน้ามากขึ้น ก็จะทำให้จำนวน component และ module โตตามไปด้วย โดยปกติแล้ว เมื่อเราเปิดแอปพลิเคชันขึ้นมา ทุกไฟล์ .js หรือ .ts ที่เรา include เข้ามา จะถูกโหลดเข้ามาทั้งหมดก่อน แต่ในบางครั้ง ฟังก์ชันหลายฟังก์ชัน ก็ยังไม่ถูกเรียกใช้งานขณะนั้น ดังนั้น ในเมื่อพัฒนาแอปพลิเคชันแยกเป็น component อยู่แล้ว เราก็ควรเรียกใช้งานเฉพาะ component ที่จำเป็นขณะนั้น เพื่อให้แอปพลิเคชันของเราทำงานได้เร็วที่สุด

Lazy Loading ดีอย่างไร

  • แอปพลิเคชันทำงานเร็วขึ้น
  • กินทรัพยากร หน่วยความจำเครื่องลดลง
